Google Analytics guidelines for Campaign Websites published via absolut.com

Last update: 2014-10-30

For Campaign Websites published via the absolut.com domain, all pageviews are tracked by default - in the TAC Analytics property. This is managed via Google Tag Manager.

If you want to see the statistics for your campaign web, ask your contact for a filtered view - which TAC will grant you access to.

If there are additional interactions that you want to track that are not pageviews, such as interactions in a JavaScript driven page, use below function calls.

Virtual Pageview

If an interaction causes JavaScript to change a majority of the web page, this can be considered a virtual pageview. To tell Google Tag Manager - and indirectly Google Analytics about this, perform this function call:

dataLayer.push({'event': 'virtualPageview', 'virtualUrl': '/se/thecampaignweb/newsletter/thankyou'});

The first part should correspond to the current location of the campaign web.

The second part can be used freely to describe the virtual page.

Interaction (Event)

If an interaction does not cause a majority of the web page to change, this is considered an event. Use this function call:

dataLayer.push({'event': 'interaction', 'eventCategory': 'thecampaignweb', 'eventAction': 'header', 'eventLabel': 'login'});

The first element should be your name for the campaign web. This should never vary within a campaign.

The second element can be used freely, ideally this should generally describe the interaction.

The third element can be used freely, ideally this should specifically describe the interaction.

Non Interaction (Event)

If it is of interest to record an occurrence that is not caused by a user interaction, such as a certain banner showing on page load, use this function call:

dataLayer.push({'event': 'noninteraction', 'eventCategory': 'thecampaignweb', 'eventAction': 'banner', 'eventLabel': 'product on black background'});

The first element should be your name for the campaign web. This should never vary within a campaign.

The second element can be used freely, ideally this should generally describe the interaction.

The third element can be used freely, ideally this should specifically describe the interaction.

It is important that you make sure that no virtual pageviews or events trigger automatically on arrival to the campaign web. This will result in incorrect bounce rates since a pageview is already triggered by default.
